### 3. Domain Controller Enumeration

Performed detailed scan of the Domain Controller at 192.168.10.10.

**Domain Information:**
- **Domain Name:** lab.local
- **DC Hostname:** WIN-LUAK1VFNJ8D
- **Operating System:** Windows Server 2022

**Services Discovered:**

| Port | Protocol | Service | Security Notes |
|------|----------|---------|----------------|
| 53 | TCP/UDP | DNS | Domain DNS server |
| 88 | TCP | Kerberos | Authentication service - target for Kerberoasting |
| 389 | TCP | LDAP | Directory services - enumeration target |
| 445 | TCP | SMB | **Signing Required** - prevents relay attacks |
| 3268 | TCP | Global Catalog | LDAP for forest-wide searches |

**Key Security Finding:**
SMB signing is required on the Domain Controller. This is a good security configuration that prevents SMB relay attacks (e.g., ntlmrelayx). Will need to focus on other attack vectors.

## Defense Perspective

### What a Defender Would See
- Network scans generate traffic patterns detectable by IDS/IPS
- Port scans create multiple connection attempts in logs
- Kerberos and LDAP enumeration may trigger security events

### Recommended Detections
- Monitor for sequential port connection attempts (port scan signature)
- Alert on LDAP queries from non-domain-joined machines
- Track failed authentication attempts from unknown IPs

### Positive Security Observations
- SMB signing enabled on DC (prevents relay attacks)
- Standard AD services properly configured

---

## Challenges Encountered

1. **Initial Network Isolation**
   - Kali was on wrong network segment initially
   - Solution: Changed VMware network adapter to VMnet10

2. **DNS Resolution**
   - Kali cannot resolve domain names (lab.local)
   - Solution (for next session): Configure /etc/resolv.conf to use DC as DNS

---

## Key Learnings

1. **Network Segmentation Matters:** The attacker machine must be on the same network segment as targets for direct communication
2. **SMB Signing:** When enabled, prevents NTLM relay attacks - need alternative attack paths
3. **Service Enumeration:** Standard AD ports (88, 389, 445, 3268) reveal this is a Domain Controller
4. **DNS Dependency:** AD environments heavily rely on DNS - controlling DNS resolution is important for attacks
