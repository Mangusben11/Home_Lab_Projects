### 2. Kali Network Configuration Fix

**BLOCKER RESOLVED:** Fixed the DNS and network persistence issues identified in the 2025-11-21 session.

#### Problem Identified
Initial attempt to configure DNS failed with "network unreachable" error because the network interface wasn't configured with an IP address yet.

#### Solution Applied
Configured network interface FIRST, then DNS was automatically set:

```bash
# Configure static IP with persistent settings
sudo nmcli con mod "Wired connection 1" ipv4.addresses 192.168.10.50/24
sudo nmcli con mod "Wired connection 1" ipv4.gateway 192.168.10.1
sudo nmcli con mod "Wired connection 1" ipv4.dns 192.168.10.10
sudo nmcli con mod "Wired connection 1" ipv4.method manual

# Activate the connection
sudo nmcli con up "Wired connection 1"
```

**Result:** Connection successfully activated. Network configuration is now persistent across reboots.

## Challenges Encountered

### Challenge 1: Network Unreachable Error

**Issue:** Initial DNS configuration failed with error:
```
;; UDP setup with 192.168.10.10#53(192.168.10.10) for lab.local failed: network unreachable.
```

**Root Cause:** Attempted to configure DNS resolver before the network interface had an IP address. The interface `eth0` was up but had no IP assigned, and no default route existed.

**Solution:**
1. Configure network interface with static IP first
2. DNS configuration automatically applied via NetworkManager
3. Proper order: IP → Gateway → DNS → Activate connection

**Lesson:** Always verify network interface configuration (`ip a`) before assuming DNS issues.

**Time Spent Troubleshooting:** ~3 minutes

---

## Key Learnings

1. **Configuration Order Matters:** Network interface must be configured with an IP address before DNS resolution can work. The error "network unreachable" indicates no route to the DNS server, not a DNS misconfiguration.

2. **NetworkManager vs Manual Configuration:** Using `nmcli` for persistent configuration is more reliable than manually editing `/etc/resolv.conf`. NetworkManager automatically regenerates resolv.conf with the correct nameserver when the connection is activated.

3. **Persistent Network Configuration:** The `nmcli` method with `ipv4.method manual` ensures settings survive reboots, unlike temporary `ip addr add` commands used in the previous session.

4. **Documentation as Infrastructure:** Adding credential management and hardware integration guides early establishes good operational security habits and prepares for lab expansion.
